## Communication Plan Summary

### Regular Communications

| Communication Type | Audience | Frequency | Owner | Channel | Next Scheduled |
|-------------------|----------|-----------|-------|---------|----------------|
| Weekly Status Update | Core team + PM | Weekly | PM | Email + Slack | Every Friday |
| Monthly Stakeholder Review | Stakeholder Committee | Monthly | PM | Meeting + slides | Last Friday of month |
| Release Announcement | All stakeholders | Per release | Release Manager | Email | Per release plan |
|                    |          |           |       |         |                |

### Ad-Hoc Communications

- **Decision Requests**: Sent via email with 2-day response SLA to Stakeholder Committee
- **Escalations**: Immediate notification to PM, then to appropriate stakeholder level
- **Change Notifications**: Change Manager coordinates, sent 1 week before change
- **Incident Alerts**: Immediate notification to on-call and stakeholder liaison

## Role-Specific Engagement Notes

### Project Manager
- Primary stakeholder liaison
- Owns regular status communications
- Escalates blockers and decision requests

### Change Manager
- Coordinates change-related stakeholder communications
- Ensures stakeholders are prepared for changes
- Manages change approval process with Stakeholder Committee

### User Advocate
- Represents end-user stakeholder group
- Provides user feedback to project team
- Validates user-facing communications

### Release Manager
- Communicates release schedules and impacts
- Coordinates deployment notifications with stakeholders
- Provides post-release reports
